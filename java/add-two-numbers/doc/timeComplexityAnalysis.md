## 时间复杂度分析

## 单次运算分析

### 算法上下界

|  算法      | constructor()     | getProp()          | setProp()           |
|------------|-------------------|--------------------|---------------------|
|  直接同步  | O(n)              | O(1)               | O(n)                |
|  需时同步  | O(1)              | O(1) ~ O(n)        | O(1)                |

- 注1： 变量n具体是指这次更新链表的链表长度


## 多次运算分析

### 模型抽象

`digitStringList`多次操作可以抽象为:

```
            <-- get操作组0 -->  <-- set操作组0 -->
construct0, [get00, get01 ...], [set00, set01 ...] ...
```

- 注1： 其中`get()`操作和`set`操作组之间顺序可以任意组合，每组包含操作可以有任意多个

### 分析过程

总的时间成本: `fn(n) = c0 + ∑(g01 + ... + g0n) + Σ(s00 + ... + s0n) +  ... `

- 注1：`fn(n)`中的`n`是总的操作的个数
- 注2： 其中变量`c0`指第一次构造函数时间，`g00`是第零组第零次操作的时间, `s00`是第零组第零次的时间
- 注3： 一个变量的生命周期只会存在一次构造过程，构造时间就是`c0`
